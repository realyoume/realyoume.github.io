OSI 七层模型

  • 应用层 为计算机用户提供服务
  • 表示层 数据处理（编码解码，加密解密，压缩解压）
  • 会话层 管理应用程序之间的会话
  • 传输层 为两个主机进程之间的通信提供通用的数据传输服务，端对端
  • 网络层 路由和寻址（决定数据在网路中的游走路径）
  • 数据链路层 帧编码和误差纠正控制
  • 物理层 透明传输比特流

TCP/IP 四层模型

  1. 应用层
  2. 传输层
  3. 网络层
  4. 网络接口层

协议

应用层协议 :

  • HTTP 协议（超文本传输协议，网页浏览常用的协议）
  • DHCP 协议（动态主机配置）
  • DNS 系统原理（域名系统）
  • FTP 协议（文件传输协议）
  • Telnet协议（远程登陆协议）
  • SSH
  • 电子邮件协议等（SMTP、POP3、IMAP）

传输层协议 :

  • TCP 协议
  • UDP 协议

网络层协议 :

  • IP 协议（TCP/IP 协议的基础，分为 IPv4 和 IPv6）
  • ARP 协议（地址解析协议，用于解析 IP 地址和 MAC 地址之间的映射）
  • ICMP 协议（控制报文协议，用于发送控制消息）
  • NAT 协议（网络地址转换协议,内网地址映射到外网地址）
  • RIP 协议、OSPF 协议、BGP 协议（路由选择协议）

网络接口层 :

  • 差错检测技术
  • 多路访问协议（信道复用技术）
  • CSMA/CD 协议
  • MAC 协议
  • 以太网技术
  • ARP

TCP　UDP

区别

1.　面向连接
1.　可靠传输
1.　有无状态
1.　效率
1.　首部开销

使用场景

  1. UDP 即时通信，如直播，视频，对速度要求高
  2. TCP 对准确性要求高的场景，如文件传输、远程登录

HTTP 基于 TCP 还是 UDP？

1.0 1.0 2.0 TCP

3.0 UDP QUIC

解决队头阻塞问题

HTTP/2 在单个 TCP 连接上使用了多路复用，受到 TCP 拥塞控制的影响，少量的丢包就可能导致整个 TCP 连接上的所有流被阻塞

三次握手和四次挥手

TCP 如何保证传输的可靠性？

  1. 基于数据库传输
  2. 用序列号对数据包进行排序和去重
  3. 校验和
  4. 超时重传
  5. 流量控制
  6. 拥塞控制

TCP 如何实现流量控制？

可变大小的滑动窗口

发送方速度过快 -> 缓冲区填满 -> 数据包丢失

窗口字段 -> 控制发送窗口大小 -> 控制发送速率

双全工通信 同时是发送端和接收端

TCP 的拥塞控制是怎么实现的？

过多的数据注入到网络中 -> 网络性能变坏

网络的拥塞程度 -> 拥塞窗口 -> 发送速度

  1. 慢开始 拥塞窗口初始值为 1
  2. 拥塞避免 拥塞窗口缓慢变大 一次往返时间加一
  3. 快重传与快恢复 发送机接收到三个重复确认，立即重传这些丢失的数据段，将拥塞窗口减半，重新进入拥塞避免阶段

ARQ 协议

自动重传请求 错误纠正协议

确认+超时 在不可靠的网路信道上实现可靠的信息传输

停止等待 ARQ 协议

每发送一个分组就等待确认

超时重传

接收方收到重复分组丢弃并确认

发送方收到重复确认丢弃

连续 ARQ 协议

维持一个发送窗口 将发送窗口内的分组连续发送出去

采用累计确认 确认最后一个分组

优点：信道利用率高

缺点：不能知道哪些分组已正确接收，可能出现回退N的情况

从输入URL 到页面展示到底发生了什么？

  1. DNS 解析（如果有本地缓存就不用）
  2. 建立TCP 连接
  3. 发送 HTTP 请求
  4. 服务器处理请求并返回 HTTP 报文
  5. 浏览器解析渲染页面
  6. 连接结束

HTTP 状态码

状态码 含义
200 OK 成功处理
201 Created 成功处理并创建资源
202 Accepted 服务端已经接收到请求，但是还未处理
301 Moved Permanently 资源被永久重定向
302 Found 资源被临时重定向
400 Bad Request 发送的请求存在问题
401 Unauthorized 没有认证
403 Forbidden 请求被拒绝
404 Not Found 找不到资源
500 Internal Server Error 服务器内部错误
502 Bad Gateway 网关错误

HTTP Header 中常见的字段有哪些？

字段头 说明
Accept 能够接受的回应内容类型
Accept-Charset 能够接受的字符集
Accept-Language 能够接受的回应内容的自然语言列表
Authorization 认证信息
Cache-Control 缓存控制
Connection 连接控制
Date 时间
Range 偏移量
If-Modified-Since 缓存是否被修改
User-Agent 浏览器身份标志

HTTP HTTPS

  1. 端口号 80 443
  2. URL前缀
  3. 安全性和资源消耗
    • 对称加密
    • 非对称加密
    • 数字签名 Certificate Authority CA 散列 加密

HTTP1.0 HTTP1.1

  1. 短连接 长连接

  2. 新增状态码

    100 请求大资源预热 206 范围请求

  3. 缓存处理

    If-Modified-Since,Expires

    Entity tag If-Unmodified-Since, If-Match

    更多缓存头和缓存控制策略

  4. 带宽优化

    不支持断点重传

    引入 range 头域，可以传输对象的一部分，206，自由灵活，节约带宽

  5. Host头

    实现虚拟主机技术，统一服务器运行多个服务，根据 Host 头决定选哪一个

HTTP/1.1 和 HTTP/2.0 有什么区别？

  1. IO 多路复用

    在同一连接上处理多个请求

  2. 二进制帧

    使用二进制帧进行数据传输，减少了传输的数据量和带宽消耗

  3. 头部压缩

    压缩 Header，减少了网络开销

  4. 服务器推送

    将资源一并推送到客户端，减少请求次数

HTTP/2.0 和 HTTP/3.0 有什么区别？

  1. 传输协议

    TCP

    QUIC Quick UDP Internet Connections

    提供与 TLS/SSL 相当的安全性，具有较低的连接和传输延迟

  2. 连接建立

    三次握手

    第一次就发送数据

  3. 队头阻塞

    HTTP/2.0 多请求复用一个 TCP 连接，一旦发生丢包，就会阻塞住所有的 HTTP 请求

    建立多个数据流，互相不受影响

  4. 错误恢复

    都有错误恢复机制

  5. 安全性

    基于 SSL/TLS 协议

    基于 QUIC 协议，内置了加密和身份验证机制，更高的安全性

HTTP 是不保存状态的协议, 如何保存用户状态?

cookie 本地保存，每次请求时携带，容量4KB，不安全，跨站请求伪造问题

session 服务端保存，容量大，用 sessionId 访问，方便管理，但占用服务端资源

实践：redis 数据库保存 session,cookie 中携带 sessionID,如果 cookie 被禁，可以在 URL 中携带 sessionID

URI 和 URL 的区别是什么?

URI 统一资源标识符，用于标识

URL 统一资源定位符,用于标识和定位，可看作 URI 的一种实现，给出了找到资源的路径

应用层协议，基于 UDP，端口 53

域名 映射到 ip地址

浏览器本地 Host 存在就不用查询

迭代查询 递归查询

  1. 根服务器
  2. 顶级域服务器
  3. 权威服务器
  4. 本地服务器

IP

IP 网路协议

最重要的协议之一，网络层协议

定义数据包的格式，对数据包进行路由和寻址

IPv6

  1. 128 位，更大的地址空间
  2. 无状态地址自动配置，自动生成全局唯一地址

NAT

局域网地址 映射到 公网地址

解决 IPv4 地址空间不足的问题

一般是通过路由器连接公网，将 ip 映射到不同端口以访问公网

socket

是一个位于运输层和应用层之间的抽象层

作为运输层的调用接口，方便我们使用运输层的服务

像文件一样的打开、读写和关闭等操作，发送和接收数据，与网络中其他设备通信

ARP

地址解析协议（Address Resolution Protocol）

IP 地址 -> MAC 地址

连接网络层和链路层

MAC 地址

媒体访问控制地址，每个网络设备都有唯一的 MAC 地址

全1地址，广播

ARP 表

IP 地址，MAC 地址，生存周期（默认20分钟）

同一局域网

先看表，没有再查询

ARP 查询分组，广播问询

ARP 响应分组，单播相应

源IP 目标IP 源MAC 目标MAC

不同局域网

发送给目标路由器（连接两个子网的）A接口

路由器查询转发表转发到B接口

B接口完成同一局域网内发送
