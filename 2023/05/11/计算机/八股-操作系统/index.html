八股-操作系统

[2023年5月11日]

什么是操作系统？

  1. 本质上是一个程序，帮助我们管理计算机软硬件资源，是计算机系统的基石
  2. 屏蔽了底层硬件复杂性
  3. 内核是操作系统的核心，负责内存管理，硬件管理，文件管理，进程管理
  4. 内核是连接应用程序和硬件的桥梁，决定着系统的性能和稳定性

核心功能

  1. 进程和线程管理
  2. 存储管理（内存，外存）
  3. 文件系统管理
  4. 设备管理
  5. 网络管理
  6. 安全管理

用户态和内核态

CPU 执行代码的两种状态，有不同的权限

用户态：用户的应用程序处于这一状态，具有较低的权限，只能读取应用程序的数据

内核态：最高的权限，可用访问任意资源，一般是执行系统代码

用户态切换到内核态

  1. 系统调用
  2. 中断
  3. 异常

什么是系统调用

系统调用是内核提供的接口，方便用户程序调用，在内核态下完成对重要资源的操作。

进程 线程

进程

一个正在运行的程序实例

是操作系统进行资源分配和管理的基本单位

不同进程独立程度高，有利于管理

进程创建、切换开销大

线程

是进程中的执行单位

一个进程可以有多个线程

不同线程可以共享资源，不利于管理

创建、切换开销小

进程有哪几种状态?

  1. 创建状态
  2. 就绪状态
  3. 运行状态
  4. 阻塞状态
  5. 结束状态
  6. 就绪挂起态
  7. 阻塞挂起态

线程同步方式

  1. 互斥锁
  2. 读写锁
  3. 信号量
  4. 屏障
  5. 事件 Wait/Notify

PCB

Process Control Block 即进程控制块,是操作系统中用来管理和跟踪进程的数据结构

当操作系统创建一个新进程时，会为该进程分配一个唯一的进程 ID，并且为该进程创建一个对应的进程控制块。

PCB 中包含进程运行的信息和状态，操作系统根据这些信息来管理和调度进程

进程间的通信方式

  1. 管道/匿名管道
  2. 有名管道
  3. 信号
  4. 消息队列
  5. 信号量
  6. 共享内存
  7. 套接字

进程的调度算法

  1. FCFS 先来先服务
  2. SJF 短作业优先
  3. 时间片轮转算法
  4. 多级反馈队列
  5. 优先级调度

僵尸进程和孤儿进程

僵尸进程：

子进程已经终止，父进程没有调用 wait()或 waitpid()等系统调用来获取子进程的状态信息，释放子进程占用的资源，导致子进程的 PCB 依然存在于系统中

孤儿进程：

一个进程的父进程已经终止或者不存在，但是该进程仍在运行。

什么是死锁

多个进程/线程被阻塞，它们都在等待一个或多个资源被释放。因为被无限期的阻塞，所以程序不能正常终止。

死锁的四个必要条件

  1. 互斥
  2. 占有并等待
  3. 非抢占
  4. 循环等待

解除死锁的方法

  1. 预防 破坏必要条件
  2. 避免 银行家算法
  3. 检测
  4. 解除

死锁的预防

  1. 静态分配策略

    申请到全部资源再执行 不会出现占有并等待的情况

    严重降低资源利用率

  2. 层次分配策略

    按照一定顺序申请和释放资源 不会出现循环等待

死锁的避免

银行家算法

尝试分配 -> 是否安全

要进行安全性检查 占用资源

死锁的检测

检测系统是否处于死锁状态

进程-资源分配图

无环路 -> 无死锁

有环路+仅有一个资源 -> 死锁

有环路+多个资源 -> 如果能在环中找到非阻塞的进程，可以消除边，如果所有的边都能消除，就不存在死锁

死锁的接触

  1. 结束所有进程，重启操作系统
  2. 撤销涉及死锁的所有进程
  3. 逐个撤销涉及死锁的进程，直到死锁解除
  4. 抢占资源

内存管理介绍

  1. 内存分配与回收
  2. 地址转换
  3. 内存扩充

内存管理机制

  1. 块式管理 固定大小的块
  2. 页式管理 物理划分
  3. 段式管理 逻辑划分
  4. 段页式管理

页表 快表 多级页表

页表 地址转换 速度慢 占用空间大

快表，页表的 cache，解决映射速度慢的问题，快表存放在高速缓冲存储器中 空间换时间

多级页表 采用分级方式，将一部分页表移除主存 时间换空间

分段 分页

共同点：

  • 提高内存利用率，减少内存碎片
  • 在内存中离散存储，但段/页中的存储是连续的

区别：

  • 页的大小固定，段的大小取决于程序
  • 分页是满足操作系统逻辑管理的要去，分段体现了逻辑划分

逻辑地址 物理地址

物理地址 真实物理内存的地址

逻辑地址 操作系统为方便编程，为程序提供的一致的、连续的的地址，通过地址转换变成物理地址

虚拟地址空间

CPU 通过 MMU（内存管理单元）实现 虚拟地址 -> 物理地址

优点：

  1. 访问物理内存中离散的空间
  2. 隔绝不同进程的内存
  3. 为虚拟内存提供实现基础

什么是虚拟内存

  1. 扩展 将内存扩展到硬盘上，让程序拥有更大的内存空间
  2. 独享 为每个进程提供一致的、私有的地址空间，产生一种独享主存的感觉

编写大程序更容易 更好利用了主存

局部性原理

  1. 时间局部性

    访问过的代码和数据可能再次访问 循环

  2. 空间局部性

    访问过的存储单元附近的存储单元也可能被访问 指令顺序存储 数据聚簇存储

虚拟内存技术实现

  1. 请求分页存储管理 请求调页+页面置换
  2. 请求分段存储管理 请求调段+分段置换
  3. 请求段页式存储管理

一定量的内存和外存 缺页中断 虚拟地址空间

页面置换算法

  1. OPT 最佳页面置换算法
  2. FIFO 先进先出页面置换算法
  3. LRU 最近最久未使用页面置换算法
  4. LFU 最少使用页面置换算法

文件系统主要做了什么？

负责管理和组织计算机存储设备上的文件和目录

  1. 存储管理
  2. 文件管理
  3. 目录管理
  4. 文件访问控制

硬链接 软链接

硬链接：

通过 inode 号建立连接，拥有同等地位

可以防止重要文件被删除

不能跨越文件系统

软链接：

指向文件路径，相当于快捷方式

可用跨域文件系统

磁盘调度算法

  1. 先来先服务算法
  2. 最短寻道时间优先算法
  3. SCAN
  4. CSCAN
  5. LOOK
  6. CLOOK
