用法细节

  1. 数字采用IEEE754双精度浮点数存储，会产生误差，解决方法是转换成整数比较，或者差值在一定范围内就认为相等。
  2. 不要用==，！=，要用===，！==，同时进行类型的比较

  4. 任何值都可作为布尔值，false: 0, -0, 0.0, NaN, “”, null, and undefined ，true: anything else

1.x

Header 不像 body，它不会被压缩。

两个报⽂之间的 header 通常⾮常相似，但它们仍然在连接中重复传输。

⽆法复⽤。当在同⼀个服务器打开⼏个连接时：TCP 热连接⽐冷连接更加有效。

2

HTTP/2 是⼆进制协议⽽不是⽂本协议。不再可读，也不可⽆障碍的⼿动创建，改善的优化技术现在可被实施。

这是⼀个复⽤协议。并⾏的请求能在同⼀个链接中处理，移除了 HTTP/1.x 中顺序和阻塞的约束。

压缩了 headers。因为 headers 在⼀系列请求中常常是相似的，其移除了重复和传输重复数据的成本。

其允许服务器在客户端缓存中填充数据，通过⼀个叫服务器推送的机制来提前请求。

3

HTTP/3基于UDP协议重新定义了连接，在QUIC层实现了⽆序、并发字节流的传输，解决了队头阻塞问题（包括基于QPACK解决了动态表的队头阻塞）

HTTP/3重新定义了TLS协议加密QUIC头部的⽅式，既提⾼了⽹络攻击成本，⼜降低了建⽴连接的速度（仅需1个RTT就可以同时完成建链与密钥协商）

HTTP/3将Packet、QUIC Frame、HTTP3 Frame分离，实现了连接迁移功能，降低了5G环境下⾼速移动设备的连接维护成本

常用方法

  1. GET ：从服务器获取URL对应的资源
  2. HEAD ：除了服务器响应中不能包含消息体，该⽅法与GET⼀样。⽤于只需少数元信息的情况
  3. POST ：被设计⽤来注解、修改URL所对应的资源
  4. PUT ：被设计⽤来修改或创建资源。当URL对应的资源存在时，则提交的作为新版本，否则新建资源
  5. DELETE ：被设计⽤来删除URL对应的资源
  6. TRACE ：主要⽤来测试。服务器将最终接收到的请求本身发送回来，作为客户端诊断依据
  7. OPTIONS ：客户端查询服务器对与某URL允许的通信选项
  8. CONNECT ：保留的⽅法名，⽤于代理切换隧道

前端鉴权

认证(Identification) 是指根据声明者所特有的识别信息，确认声明者的身份。

授权(Authorization)： 在信息安全领域是指资源所有者委派执⾏者，赋予执⾏者指定范围的资源操作权限，以便对资源的相关操作。

鉴权(Authentication) 在信息安全领域是指对于⼀个声明者所声明的身份权利，对其所声明的真实性进⾏鉴别确认的过程。

权限控制(Access/Permission Control) 将可执⾏的操作定义为权限列表，然后判断操作是否允许/禁⽌

DNS

Domain Name System 域名系统

作用：

正向解析：域名查找ip地址

反向解析：ip地址查找域名

DNS服务器构成一个树的结构，根据域名的构成来组织

查询时有递归查询和迭代查询

缓存

  • 为了减少DNS流量，名称服务器会缓存域名/IP地址映射信息
  • 当查询的条⽬在缓存中时，服务器不会联系其他服务器

URL/URI/URN

URI, Uniform Resource Identifier，统⼀资源标识符, ⽤于标识某⼀互联⽹资源名称的字符串

URL, Uniform Resource Locator, 统⼀资源定位符,它标识⼀个互联⽹资源，并指定对其进⾏操作或获取该资源的⽅法。

URN, Uniform Resource Name, 统⼀资源名称, 仅⽤于命名，⽽不指定地址

URN和URL可以看作URI的不同实现。
