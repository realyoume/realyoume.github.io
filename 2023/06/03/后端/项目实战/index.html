项目结构

每个模块都相当于一个小项目

xuecheng-plus-parent

所有模块的父模块，主要用于版本管理

通过 dependencyManagement 锁定项目依赖的版本

xuecheng-plus-base

包含一些通用的代码

其他模块通过 maven 引入

xuecheng-plus-content

具体的功能模块

MybatisPlus

可以根据数据库结构生成实体类，持久层，业务层，接口层代码

分页查询

占位符

#{} 占位符，可以防止 sql 注入

${} 动态拼接字符串，会导致 sql 注入

Swagger

用于编写接口文档

可以读取代码生成文档，并直接测试

数据字典表

用于定义状态与状态码之间的映射关系

方便之后的修改和维护

单元测试

每写一个类就要有一个单元测试

HttpClient

IDEA 自带的插件，可以方便的发送请求测试接口

请求数据用文件保存，可以与项目文件同时存储

跨域

参数校验

JSR 303 校验

@RequestBody @Validated

开启对对象的参数校验

当参数非法时，会报出 MethodArgumentNotValidException 错误，并携带信息

可以通过统一异常处理进行管理，将报错返回给前端

分组

参数校验可以分组进行

同一个对象在不同情况下使用不同的校验规则

可以通过 groups 属性来配置

配置文件信息

对应文件 content-api-dev.yaml

namespace: dev

group: xuecheng-plus-project

extension-configs

扩展配置文件，拿到其他的配置文件

shared-configs

将公共的配置信息（如数据库连接，日志记录）抽取出来，其他文件通过共享的方式引入

配置优先级

  1. 项目应用名配置文件
  2. 扩展配置文件
  3. 共享配置文件
  4. 本地配置文件

spring.cloud.nacos.config.override-none 本地最优先
