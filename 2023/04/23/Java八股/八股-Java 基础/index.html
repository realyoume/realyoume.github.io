八股-Java 基础

Java 语言的特点

  1. 简单易学

    移除了指针、多继承、内存管理等概念

  2. 面向对象

    封装 继承 多态

  3. 平台无关性

    字节码 JVM 跨平台运行

  4. 支持多线程

    Thread Runnable 线程池

  5. 编译与解释并存

    源代码 -> 字节码 编译

    JVM 执行字节码 解释

Java C++

  1. Java 没有指针
  2. Java 自动内存管理，不用 alloc free
  3. Java 只有单继承
  4. Java 不支持操作符重载
  5. C++ 编译型 Java 编译与解释并存

JVM JDK JRE

JVM Java Virtual Machine Java 虚拟机 运行字节码
JDK Java Develpoment Kit Java 开发工具包 JRE+编译器+调试器+ Javadoc
JRE Java Runtime Environment Java 运行环境 JVM+Java 类库

什么是字节码?

  1. JVM 执行的代码
  2. 扩展名为 .class

好处

  1. 解释型语言 可移植性
  2. 编译与解释并存 性能好

编译型语言 解释型语言

编译型语言：编译器 代码 -> 机器码 速度快 C、C++

解释型语言：解释器 一边解释一边执行 可跨平台 Python,JavaScript

JVM 执行字节码优化措施

JIT

通常，JVM 一边解释一边执行字节码，速度慢。

just in time compiler 即时编译器，将热点代码（经常执行的代码，循环块中的代码）直接翻译成机器码并保存，极大提升了效率。

JIT 在运行时优化，需要预热

AOT

Ahead Of Time Compile，在程序启动时就将字节码翻译成机器码，避免了 JIT 的性能损耗。

标识符 关键字

标识符：一个名字

关键字：被赋予了特殊含义的标识符

成员变量 局部变量

成员变量属于类

局部变量属于方法

  1. 修饰符

    成员变量可以被 public，private，static，final 修饰，局部变量只能被 final 修饰

  2. 存储空间

    成员变量在堆或方法区，局部变量在栈中

  3. 默认值

    成员变量有默认值，局部变量没有

  4. 生命周期

    成员变量生命和类或实例保持一致，局部变量和方法的调用同步

重写 重载

重写

子类改变父类方法的实现

方法名相同，参数列表相同

返回值小于等于父类，抛出异常小于等于父类

访问修饰符大于等于父类

重载

同一个类中方法的不同表现

方法名相同

参数列表不同

返回值，访问修饰符可以不同

可变长参数

在方法中传入不定长度的参数

语法糖，基于数组实现

前面可以有其他参数，只能作为最后一个参数

System.out.println() 依靠可变长参数实现

基本数据类型

byte short int long float double char boolean

基本类型 包装类型

Java 为每个基本类型提供了包装类型，符合一切皆对象的思想，方便了很多操作。（容器，工具方法）

  1. 包装类型是对象
  2. 包装类型默认值是 null，基本类型默认值是相应零值
  3. 基本类型占用更少的空间

包装类型 缓存机制

Byte Short Integer Long Character Boolean

提前在数组中创建好缓存对象，通过 valueof 方法获得时直接返回

自动装箱 自动拆箱

基本类型 <-> 包装类型

语法糖

XXX.valueOf()

X.xxxValue()

float double

类型 符号位 指数 尾数 长度
float 1 8 23 32
double 1 11 52 64

科学计数法 二进制表示 有精度丢失风险

BigDecimal

推荐构造方法 BigDecimal(String val) BigDecimal.valueOf(double val)

divide 指定保留位数，保留规则

equals 同时判断值和位数

compareTo 只判断值

面向对象 面向过程

面向过程：将问题的分解成一个个方法，通过方法的执行来完成。

面向对象：将问题通过对象的的行为和交互来完成。

一般来说，面向对象的程序具备可重用、易维护、易扩展特性

封装

属性和方法组合成有机整体

将类的信息隐藏在类的内部，外部只能通过提供的接口进行访问，提高了类的安全性和可维性

继承

使用一个类为基础来构造另一个类

重用代码，提高可维护性，提高开发效率

  1. 子类拥有父类全部属性和方法
  2. 子类可以任意扩展属性和方法
  3. 子类可以重写父类方法

多态

一个对象具有多种状态，具体实现是父类引用指向子类实现

引用类型和实现类型之间具有继承、实现关系

在运行时确定执行的方法

接口 抽象类

相同点：

  1. 不能实例化
  2. 包含抽象方法
  3. 可以有默认实现（接口 default）

不同点：

  1. 接口用于体现一种规范，抽象类用来代码复用

  2. 接口 方法 public abstract 属性 public static abstract

    抽象类 任意

  3. 单继承 多实现

引用拷贝 浅拷贝 深拷贝

引用拷贝：直接复制引用

浅拷贝：堆中创建对象，属性如果是对象，指向原属性

深拷贝：堆中创建对象，属性如果是对象，同样创建

Object

  1. getClass
  2. hashCode
  3. equals
  4. clone
  5. toString
  6. wait
  7. notify
  8. notifyAll
  9. finalize

HashCode

内存地址通过哈希算法得到的一个 int 整数

用于快速判断两个对象是否相等

与 equals 配合使用

满足条件

hashcode 相同，不一定 equals

equals，hashcode 一定相同

String StringBuilder StringBuffer

都是用 byte/char 数组来保存

String 不可变，每次修改产生新对象，线程安全

StringBuilder 可变，提供修改方法，线程不安全，性能略优

StringBuffer 可变，提高修改方法，线程安全

String 数组实现

Java8 之前用 char 数组存储

Java9 开始用 byte 数组存储

目的：很多字符串只包含拉丁字符，可以用 byte 存储，节省一半空间

String 不可变

目的

String 的使用场景广泛，将 String设置为不可变可以提高效率和安全性

实现

  1. 字符数组被 private final 修饰，且不提供修改的方法
  2. String 类被 final 修饰，包装不被子类破坏

字符串常量池

保存字符串常量，避免重复创建

字符串不可变，可以持有引用

从永久代 移动到 堆中

StringTbale 保存引用

intern() 将字符串保存到常量池并返回引用，如果已存在，直接返回引用

异常

Throwable 可被抛出

  • Error 错误，如OOM，一半会直接终止

  • Execption

    • Checked Execption 受检查异常，必须处理，否则不通过编译

      如IO相关异常

    • Unchecked Execption 不受检查异常，可以不用处理

      空指针异常，数组越界异常，算数错误

try with resources

  1. 任何实现 java.lang.AutoCloseable或者 java.io.Closeable 的对象
  2. 在 try-with-resources 语句中，任何 catch 或 finally 块在声明的资源关闭后运行

泛型

声明对象类型，JDK5 新特性，增强代码可读性和稳定性

传入对象时检查对象类型

得到对象时转化对象类型

  1. 泛型类
  2. 泛型接口
  3. 泛型方法

反射

在运行时得到一个类的属性和方法，并可以使用这些属性和方法

应用场景：大量运用在框架中

  1. 动态代理
  2. 注解

优点：增加了灵活性，为框架提供了便利

缺点：存在安全问题，比如可以绕开泛型的安全检查（编译期 运行时）

获得class四种方法：

  1. 通过类获得
  2. 通过实例获得
  3. class.forName() 全类名获得
  4. 通过类加载器传入类路径获得

注解

一种特殊的注释，修饰类，方法，变量

提供某种信息供编译或运行时使用

本质是是一个继承了 annotation 的接口

我们也可以自定义注解

作用方式

  1. 编译期直接扫描，原生注解
  2. 运行时通过反射处理，如框架中的一些注释

SPI API

都是接口，作为一种规范，方便模块之间通信

解耦，可扩展性，可维护性

区别：

服务提供者定义的接口 API

服务使用者定义的接口 SPI

序列化

目的：持久化对象、网络传输对象

Java 对象/数据结构 <-> 二进制字节流

static 属于类，不属于对象，不会被序列化

transient 修饰变量，不会被序列化

文本类序列化：可读性好，如XML，JSON

二进制序列化：性能好，hession,kryo,Protobuf,ProtoStuff

语法糖

增加代码可读性，让代码更加简洁

语法糖会在编译阶段被还原成基础的语法，这个过程被称为解糖

场景语法糖

  1. 增强for循环
  2. 泛型
  3. 可变长参数
  4. try with resources
  5. 自动拆装想
  6. 数字字面量

值传递 引用传递

值传递：创建拷贝，对形参修改不会影响实参

引用传递：传递引用，对形参修改会影响实参

Java 只有值传递，基本类型拷贝值，对象类型拷贝地址。

代理模式

使用代理对象代替原有对象

在不改变原有对象的基础上扩展其功能
